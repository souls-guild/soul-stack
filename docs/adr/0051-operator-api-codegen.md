# ADR-051. Operator API codegen: OpenAPI → Go-типы (oapi-codegen), types-only → strict

> **Статус: SUPERSEDED [ADR-054](0054-openapi-code-first.md) (2026-06-12), реализация СНЕСЕНА (2026-06-13, HEAD `fde65bf`).** Посылка инвертирована: было **OpenAPI → Go-типы** (spec-first, oapi-codegen), стало **Go-типы → OpenAPI** (code-first, huma v2). Решение пользователя. **Тираж завершён, каркас oapi-codegen этого ADR ПОЛНОСТЬЮ удалён из кода:** рукописная `openapi.yaml` как источник, пакет `keeper/internal/api/oapi/` (committed `types.gen.go`/`server.gen.go`), мост `oapi_strict.go`/`strictServer`, oapi-codegen-тулчейн и Makefile-таргеты `gen-api`/`check-gen-api`/`sync-openapi` — снесены. Источник формы теперь — Go-типы huma (см. [ADR-054](0054-openapi-code-first.md)). Текст ниже описывает уже снятый spec-first-подход и остаётся только историческим контекстом перехода.

> **Историческая справка (до supersede): Фаза 1 (`types-only`) ЗАВЕРШЕНА (2026-06-09/10), в baseline.** Решения пользователя + дизайн architect залочены и реализованы. Все ~21 wire-хендлеров Operator API мигрированы на сгенерённые типы `keeper/internal/api/oapi` (oapi-codegen **v2.7.1**, pinned): gen-target, `types.gen.go`, тираж alias/конвертеров (категории A–D), удаление `proto/operator/v1`, даунгрейд спеки до `3.0.3` — все сделаны. Закрыты инварианты strict-цели (1) единый источник формы и (3) идемпотентность кодогена (d). **Фаза 2 (`types-only → strict`, инвариант (2) — валидация формы кодогеном, слайс S6) — следующий шаг** по явному решению.

**Контекст.** [ADR-011](0011-go-layout.md#adr-011-раскладка-go-кода-gowork-с-модулями-по-сторонам) положил источником **формы** Operator API пакет `proto/operator/v1/` (6 `.proto`, messages для 16 HTTP-эндпоинтов). Но транспорт Operator API — **HTTP/JSON** ([ADR-004](0004-binaries.md#adr-004-раскладка-бинарей--keeper-soul-soul-lint-push-режим--модуль-внутри-keeper)), не gRPC; gRPC-сервиса над `operator/v1` нет. В итоге `proto/operator/v1` оказался **паразитной второй моделью формы** рядом с [`docs/keeper/openapi.yaml`](../keeper/openapi.yaml): рукописные DTO-типы хендлеров молча расходятся со спекой (request-drift), а спека — единственное, что реально едет на проволоку и читается web-стороной. Нужен **один источник формы** = OpenAPI-спека, из которой Go-типы генерируются, а не пишутся руками.

**Решение.**

**(a) Инструмент — oapi-codegen v2 (pinned), режим MVP `types-only`.** Генератор — `oapi-codegen` v2 с pinned-версией. Конфиг кодогена в MVP — **только модели** (`generate: { models: true }`): из спеки рождаются Go-типы request/reply, без серверного каркаса. Пакет назначения — **`keeper/internal/api/oapi`**, файл **`types.gen.go`** (committed, как proto-gen по [ADR-011](0011-go-layout.md#adr-011-раскладка-go-кода-gowork-с-модулями-по-сторонам)). Makefile получает таргет **`gen-api`** (генерация) + **`check-gen-api`** (проверка идемпотентности — паритет с существующим `check-gen` для proto). Авто-`HandlerFromMux` в MVP **не используется** (см. (d)).

**(b) Источник правды формы — `docs/keeper/openapi.yaml`.** Форма (request/reply-схемы, required, enum, format) живёт в OpenAPI-спеке; Go-типы — её производная. `proto/operator/v1` **упраздняется** (аменд [ADR-011](0011-go-layout.md#adr-011-раскладка-go-кода-gowork-с-модулями-по-сторонам), см. ниже). Семантику/mapping (endpoint ↔ MCP-tool ↔ permission, транспортные правила, error-format) по-прежнему держит [`docs/keeper/operator-api.md`](../keeper/operator-api.md) — это markdown-нормирование, не вторая модель формы.

**(c) Паттерн интеграции — type-alias к gen-типу + конвертер домен→wire на границе.** Сгенерённый тип не правится руками; код хендлеров привязывается к нему через **type-alias**, а домен переводится в wire-форму **конвертером на границе** (там же, где сериализация). Категории тиража по этому различению:
- **A — request-типы** (входящее тело): чистый **alias** к gen-типу. Хендлер декодит прямо в gen-структуру.
- **B — reply со scalar-полями** (плоские строки/числа/bool): чистый **alias**.
- **C — reply с typed-enum / pointer-optional, читаемыми тестами**: **конвертер-на-границе, НЕ alias**. Домен-структура остаётся отдельной (её ассертят тесты), на выходе мапится в gen-wire-тип. Alias тут запрещён — он бы притянул gen-форму (указатели/enum-обёртки) в домен и в тесты.
- **D — reply-поле byte-passthrough JSONB** (домен поля = `json.RawMessage`, сырые байты из PG отдаются as-is): в спеке поле помечается `x-go-type: json.RawMessage` → gen-тип поля становится `json.RawMessage` → домен **алиасится чисто** (gen-форма поля совпадает с доменом). `x-go-type-import` **НЕ добавляется**: oapi-codegen всегда импортирует `encoding/json` сам (marshal-код gen-файла), а явная import-директива дала бы дубль-импорт `json "encoding/json"` рядом с неявным → `json redeclared` (gen-файл не компилируется). Конвертер `map`↔`raw` ЗАПРЕЩЁН: PG-JSONB-канонизация ≠ Go `map`-marshal (лексикографический порядок ключей) → unmarshal→map→marshal **переупорядочивает ключи ВНУТРИ passthrough-значения** = wire-change самого значения поля (а не безобидный порядок членов объемлющего объекта — см. несущий инвариант (f)), что нарушает «ноль wire-change»: для byte-passthrough JSONB сырые байты значения и есть контракт, их перетасовка меняет значение. Отличать от обычного `type: object` map-домена (pushprovider/profile — домен `map[string]any`, gen-`map` уже совпадает, x-go-type там **НЕ** применять). Признак D: домен поля — `json.RawMessage` из нетронутого PG-JSONB. На каждое D-поле обязателен **guard-тест byte-exact** — непустой multi-key payload с НЕ-лексикографическим порядком ключей, ассерт сырых байтов passthrough-значения (не unmarshal-эквивалента); тест ловит регресс возврата `map`-конвертера. (Опционально симметричный request-вход с RawMessage-доменом тоже x-go-typed — снимает конвертер на входе, additive, wire-нейтрально.)

**Правила паттерна для тиража домена (developer применяет на каждом домене):**
- **Прежняя сериализация `time.Time` проверяется ДО миграции** — голый `time.Time.MarshalJSON` даёт наносекунды, такой домен **НЕ усекать** (см. (f)); прежний `.Format(time.RFC3339)` — секунды, усекать.
- **Initialism-набор кодогена** — `AID` / `SID` / `JWT` / `SHA256` / `PEM` (`additional-initialisms` в `oapi-codegen.yaml`). Normalizer трогает только Go-имя поля, не json-тег — поэтому пополнение набора **не меняет wire**, оно лишь схлопывает домены, у которых единственной C-причиной был normalizer-mismatch (`Sha256`/`PubkeyPem`/`Jwt`), в чистый alias.
- **Категорию C обосновывать ТОЛЬКО реальной причиной** — typed-enum или pointer-optional, читаемые тестами. Normalizer-mismatch **не** причина для C (он снимается initialism-ом). Если после регена домен совпал с gen-типом поле-в-поле — **схлопывать в alias**.
- **`omitempty`-тег — по спеке.** Домен обязан совпадать с gen-тегом. Расхождение «домен ↔ gen» — это **bug домена** (правим домен под спеку) либо **bug спеки** (propose-and-wait к architect). Намеренный wire-change (напр. sigil `revoked_at` null→omit для схождения со спекой) фиксируется guard-тестом.

**(d) Путь `types-only → strict`.** Целевое strict-состояние определяется **тремя инвариантами**: (1) спека — единственный источник формы (нет рукописных DTO-дублей); (2) `required`/`enum`/`format` энфорсятся кодогеном (strict-server middleware), а не ручными проверками; (3) кодоген идемпотентен. **MVP останавливается на `types-only`** — закрывает инв. 1 и 3; **инв. 2 (валидация формы кодогеном) — после явного решения** о переходе на strict, отдельным слайсом **S6** (не в этом эпике). Дополнительно дизайн фиксирует **три strict-инварианта** перехода:
- **Запрет авто-`HandlerFromMux`.** Mount роутов — **ручной**: авто-генератор роутера навесил бы хендлеры мимо per-route RBAC/Audit-middleware (security-регресс — потеря enforcement на эндпоинте). При переходе на strict-server роуты монтируются вручную поверх strict-обёрток.
- **Свои error-handler-ы.** `RequestErrorHandlerFunc` / `ResponseErrorHandlerFunc` кодогена перенаправляются в **`problem.Write`** (единый RFC 7807-формат, [operator-api.md → Error format](../keeper/operator-api.md#error-format-rfc-7807)) — иначе strict-слой вернёт свой не-problem+json-формат ошибки, расходясь с error-контрактом кластера.
- **Полнота спеки по query.** Для query-параметров (keyset-пагинация, фильтры) спека обязана явно задавать `style`/`explode` — иначе strict-биндинг query разойдётся с фактическим парсингом.

**(e) OpenAPI-версия / nullable — даунгрейд спеки до `3.0.3`** (выбор пользователя). Причина даунгрейда — **oapi-codegen v2 не поддерживает OpenAPI 3.1**: на заголовке `3.1.0` он печатает варнинг на каждом прогоне. Объявлять `3.1.0`, когда кодоген умеет только 3.0, — латентный риск: любая будущая 3.1-only-конструкция в спеке сломалась бы молча. Контент спеки **100% совместим с 3.0** (3.1-only-фич нет — отсутствуют `$defs`, `type: [T, null]`, `webhooks`, `const`, `prefixItems`, `unevaluatedProperties`), поэтому даунгрейд **безопасен и без потерь**. (Эмпирически проверено S1b marshal-сравнением: oapi-codegen v2.7.1 читает `nullable` **одинаково на 3.1.0 и 3.0.3** — `types.gen.go` байт-в-байт идентичен до и после даунгрейда; версия влияет только на варнинг кодогена, не на эмит. Прежняя посылка «3.1.0 ломал oapi-codegen: игнор `nullable` → Go omit» **опровергнута**.)

**Контракт nullable Go↔TS — структурно согласован двумя классами** (не требует ни даунгрейда, ни спец-обработки в тираже):
- **required + nullable** → Go эмитит `*T` **без** `omitempty` (значение `null` присутствует в JSON) = TS `T | null`. Симметрия `null ↔ null`.
- **optional + nullable** → Go эмитит `*T` **с** `omitempty` (поле может отсутствовать) = TS `T | null` с `?` (absent допустим). Go-omit ↔ TS-optional.

Расхождения для критичных полей нет. **Инвариант (уже выполнен, доказан эмпирически S1b):** Go-emit (oapi-codegen) и TS-emit (openapi-typescript на [soul-stack-web](#)) едят **одну** спеку и дают согласованную null-семантику. **Тираж НЕ нуждается в спец-моделировании nullable** — существующая форма консистентна по построению. Сам даунгрейд заголовка спеки — параллельный слайс **S1b** (правка `openapi.yaml`), не этот документ.

**Несущий инвариант миграции — «ноль wire-change» = СЕМАНТИЧЕСКАЯ эквивалентность wire.** Type-only-перевод хендлеров обязан сохранить wire-форму, но «ноль wire-change» нормируется **семантически**, а не побайтово: тот же набор ключей, те же значения, та же `null`-vs-omit-семантика поля. **Порядок членов JSON-объекта НЕ нормируется** — field-order может меняться при codegen (alias/конвертер-на-границе мапит в gen-структуру с другим порядком полей), и это **допустимо**: порядок членов JSON-объекта не семантичен (RFC 8259), web-сторона и любой корректный парсер читают объект по ключам. Где ниже встречается «байт-в-байт» / «byte-exact» (точность date-time, guard-тесты категории D) — речь о значениях и null-vs-omit конкретных полей, не о позиционном порядке членов объекта. Решение зафиксировано пользователем (2026-06-09/10).

**(f) Формат date-time на wire — RFC3339 UTC, точность СОХРАНЯЕТСЯ ПО ДОМЕНУ.** Точность date-time на проволоке **не меняется при миграции типов** — несущий инвариант: type-only-перевод = ноль изменений на проволоке (в смысле семантической эквивалентности, см. выше — значение и null-vs-omit, не порядок членов объекта). Конвертер домен→wire (c) выбирается по прежней сериализации домена:
- Домен с прежним **СЕКУНДНЫМ** wire (формат был `.Format(time.RFC3339)` или эквивалент, дробная часть уже отбрасывалась) → `.UTC().Truncate(time.Second)` (operator / service / audit / sigil).
- Домен с прежним **НАНОСЕКУНДНЫМ** wire (голый `time.Time.MarshalJSON` → RFC3339Nano, напр. choir `created_at`/`added_at`) → `.UTC()` **БЕЗ** `Truncate` — усечение сломало бы байт-в-байт.

Глобальный `Truncate(time.Second)` первой редакции (f) **ОТОЗВАН** — он противоречил инварианту «ноль wire-change» для наносекундных доменов. Однородная секундная точность по всему API — **отдельная опциональная контракт-нормализация** (намеренный wire-change), вне тиража, по явному решению.

**(g) Required-вложенные объекты на `types-only`.** Вложенный объект-поле, по смыслу обязательный, в `types-only`-режиме **НЕ помечается** `required:` в схеме: oapi-codegen emit-ит required-объект как value-тип (не указатель), что ломает nil-presence-детект, а авто-валидации `types-only` не даёт. Presence такого объекта энфорсится **вручную** — 422 `application/problem+json` в хендлере. При переходе на strict (d, инв. 2) `required` энфорсит кодоген, и этот ручной паттерн снимается.

**(h) Web-синхронизация — операционный инвариант.** [soul-stack-web](#) vendor-ит **ту же** спеку (`vendor/openapi/keeper.yaml`) и гонит её через `openapi-typescript` → `types.gen.ts`. Синк core→web — **cross-repo** и в core `make check` **не ловится**: при правке `openapi.yaml` нужен ручной vendor-update в web + `npm run gen:api`. Фиксируется как операционный инвариант (кандидат на CI-guard, **backlog**) — иначе третья копия спеки (web-vendor) тихо отстанет.

**Обоснование.**
- **Один источник формы.** Request/reply-форма перестаёт жить в двух местах (proto + спека); рукописные DTO-дубли удаляются, drift между хендлером и спекой невозможен на типах.
- **Согласованность Go↔TS.** Обе кодогенерации (Go и TS) питаются одной спекой; null-семантика симметрична по построению (e): required+nullable → `null ↔ null`, optional+nullable → Go-omit ↔ TS-optional. Wire-контракт сходится без спец-обработки — даунгрейд до 3.0.3 (e) на это не влияет (его причина — отсутствие 3.1-поддержки в oapi-codegen, а не nullable).
- **Безопасность не страдает.** Ручной mount + свои error-handler-ы (d) сохраняют per-route RBAC/Audit и единый problem+json — кодоген не подрывает существующие security-инварианты Operator API.

**Consequences.**
- **Удаление `proto/operator/v1`** (6 `.proto`) + его gen + правка Makefile gen-target (теряет operator-proto-секцию) — отдельным слайсом (аменд [ADR-011](0011-go-layout.md#adr-011-раскладка-go-кода-gowork-с-модулями-по-сторонам)).
- **Новый пакет** `keeper/internal/api/oapi/` с committed `types.gen.go`; Makefile-таргеты `gen-api` / `check-gen-api`.
- **`types-only` не даёт авто-валидации** формы (required/enum/format) до перехода на strict (d, S6) — на MVP presence/валидация местами ручные (g).
- **Третья копия спеки** (web-vendor) живёт вне `make check` (h) — операционный риск рассинхрона, кандидат на CI-guard.
- **OpenAPI / MCP / operator-api.md** — семантика и mapping остаются в [operator-api.md](../keeper/operator-api.md); спека-источник — [openapi.yaml](../keeper/openapi.yaml) (docs-writer при имплементации).

**Отвергнутые альтернативы.**
- **(а) Заголовок спеки `3.1.0`.** oapi-codegen v2 не поддерживает 3.1 — варнинг на каждый прогон + латентный риск молчаливой поломки на будущей 3.1-only-фиче. Отвергнут — даунгрейд до `3.0.3` (e). (Контракт nullable от версии заголовка **не зависел** — эмпирически S1b; это не было причиной отвержения.)
- **(б) Авто-`HandlerFromMux`.** Навесил бы хендлеры мимо per-route RBAC/Audit (security-регресс). Отвергнут — ручной mount (d).
- **(в) Сохранение `proto/operator/v1` как формы.** Вторая модель формы рядом со спекой = request-drift; gRPC-сервиса над ней нет. Отвергнут (b).
- **(г) Alias на reply-типы категории C.** Притянул бы gen-форму (pointer/enum-обёртки) в домен и тесты. Отвергнут — конвертер-на-границе (c).
- **(д) `map`-re-marshal passthrough-полей категории D.** Распаковка byte-passthrough JSONB (`json.RawMessage`) в `map[string]any` и обратный marshal на границе переупорядочивает ключи (Go-`map`-marshal лексикографичен, PG-JSONB-канонизация — нет) = wire-change порядка ключей, нарушает «ноль wire-change» (f). Отвергнут — `x-go-type: json.RawMessage` + чистый alias, байты отдаются as-is (категория D в (c)).

**Slice-план S.** S1a — канон (этот ADR + аменд ADR-011). S1b — даунгрейд `openapi.yaml` до `3.0.3` (параллельно). **Amendment (2026-06-10, S6: паттерн strict-тиража Фазы 2 — «мост», full-strict только greenfield).** oapi-codegen v2.7.1 в strict-режиме декодит тело лояльно (`json.NewDecoder` без `DisallowUnknownFields`) внутри сгенерённого `strictHandler`, без hook на способ декода; `StrictHTTPServerOptions` контролирует только error-handler-ы. Поэтому требование «unknown-поле → 400» при инварианте «ноль wire-change» несовместимо с типизированным `request.Body` strict-слоя. **Канонический паттерн тиража для существующих доменов:** `bridgeMiddleware` буферизует тело и кладёт `(w,r,body)` в `ctx`; strict-метод агрегата `strictServer` **НЕ** использует typed `request.Body`/typed-response, а достаёт `(w,r)`, перематывает Body и делегирует в существующий доменный `func(w,r)`-handler (он несёт `DisallowUnknownFields`-декод, presence-валидацию, `problem.Write` и точную wire-форму); метод возвращает no-op `handledResponse`. **Компиляторные гарантии моста:** полнота операций `StrictServerInterface` (забыть операцию нельзя), бинд path/query wrapper-ом, единые problem-error-handler-ы; panic-stub на немонтированных методах = явная карта немигрированного. **Сознательно НЕ получаем:** компиляторного enforcement типизированного входа/выхода и статус-кода (держится тестами и категориями A–D Фазы 1). **Greenfield-эндпоинты** (без legacy-handler-а и инварианта ноль-wire, напр. `POST /v1/incarnations/{name}/rerun-create`) пишутся сразу full-strict (typed `request.Body` + typed-response). Строгость «unknown-поле → 400» на greenfield добирается ручной pre-проверкой забуференного тела (`bridgeMiddleware`-буфер + `DisallowUnknownFields`, helper `strictDecodeProbe`) ДО доменной оркестрации — typed-декод oapi лоялен, но решение «unknown→400» сохраняется и для greenfield-эндпоинтов. Typed- и bridge-методы **совместимы в одном агрегате пометодно** — допускается постепенный перевод bridge→typed отдельными слайсами. Глобальный full-strict-перевод legacy = повторная Фаза 1 (re-wire + ре-доказательство date-time/omitempty per-domain) — **отвергнут пользователем** как несоразмерный; backlog при конкретной потребности. **Нормативные инварианты mount-а Фазы 2** (были в комментарии кода, поднимаются в канон): (1) запрет авто-`HandlerFromMux` — mount только ручной per-route с сохранением навески RBAC/Audit/Toll/Tempo; (2) свои `Request`/`ResponseErrorHandlerFunc` → `problem.Write`; (3) wire-семантическая эквивалентность миграции.

**Amendment (2026-06-10, S6: нормализация bind-ошибок query на 400).** Инвариант «ноль-wire-change» уточняется: bad-value typed-query-параметра (`Format: date-time`, `Type: integer/boolean`) нормализуется на **400 `TypeMalformedRequest`** во всём Operator API (bind `ServerInterfaceWrapper` как single source).
